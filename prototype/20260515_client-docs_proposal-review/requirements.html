アクター定義

フェーズ アクター 説明
全フェーズ 管理者 コンテンツ登録・アカウント発行・システム設定を担う運用担当者（喜多・吉良等）
Phase 1〜4 スタッフ（未認証） ログイン不要でURL・QRコードからアクセスする現場スタッフ
Phase 5〜 店舗管理者 店長相当。担当店舗のコンテンツ全てと店舗管理者向けコンテンツを閲覧できる
Phase 5〜 一般スタッフ 社員・バイト。権限に応じた範囲のコンテンツを閲覧できる
※ Phase 5以降、「スタッフ（未認証）」は廃止しログイン必須とする想定。管理者ロールの分割（管理者 / システム管理者）は将来検討（エンハンス参照）。

Phase 1 動画・テキストマニュアルを閲覧できる（テスト環境）

ユーザーストーリー：スタッフとして、マニュアルの動画と工程テキストをスマホ・タブレット・PCで閲覧できる。
環境：テスト環境のみ。コンテンツは直接DB投入（1〜2本のテストデータ）。本番公開はPhase 2で行う。
UC-ID ユースケース名 アクター 概要
UC-101 トップ画面を表示する スタッフ（未認証） URLにアクセスするとマニュアルのトップ画面（カテゴリ一覧）が表示される
UC-102 カテゴリ一覧を参照する スタッフ（未認証） ブランド・業務種別のカテゴリ一覧を閲覧し、目的のマニュアルに辿り着ける
UC-103 マニュアル一覧を閲覧する スタッフ（未認証） カテゴリ内のマニュアル一覧（タイトル等）を表示する
UC-104 マニュアル詳細を表示する スタッフ（未認証） 詳細ページを開くとYouTube動画と工程テキスト説明が1画面に表示される
UC-105 YouTube動画を視聴する スタッフ（未認証） 埋め込み表示されたYouTube動画を再生・停止できる
※ 各マニュアルは固定URLで直接アクセス可能（QRコードでの案内を想定。QR生成は外部ツールで対応）。スマホに最適化する（PC・タブレットでも表示可能だが、レイアウト崩れは許容）。

Phase 2 本番環境構築

目的：Phase 1のテスト環境で検証済みのシステムを本番に移行する。ホスティング・ドメイン・デプロイパイプラインを整備し、現場スタッフが実際にアクセスできる状態にする。
UC-ID ユースケース名 アクター 概要
UC-201 本番環境を構築する 管理者 ホスティング・ドメイン・デプロイパイプラインを整備し、本番公開できる状態にする

Phase 3 閲覧UX拡充

目的：現場スタッフが使いやすい閲覧体験を提供する。全画面・PDF印刷・習熟度別分類・検索を追加する。管理機能（Phase 4）より先に閲覧品質を高める。
UC-ID ユースケース名 アクター 概要
UC-301 動画を全画面表示する スタッフ（未認証） 動画再生中にボタン1つで全画面モードに切り替えられる（スマホ横持ち対応）
UC-302 印刷用PDFをダウンロードする スタッフ（未認証） 「印刷用PDFボタン」をタップするとPDFが取得でき、紙への印刷・現場貼り出しができる
UC-304 習熟度別カテゴリでマニュアルを参照する スタッフ（未認証） 「初級・中級・上級」などの習熟度軸でマニュアルをフィルタリングして参照できる
UC-305 マニュアルを検索する スタッフ（未認証） キーワードでマニュアルを横断検索し、目的のコンテンツに素早く辿り着ける

Phase 4 管理UI（コンテンツ管理）

目的：吉良さん（非エンジニア）が自分でコンテンツを登録・更新できる管理UIを導入する。これ以降、コンテンツ投入に開発者が介入する必要をなくす。将来的には店舗スタッフも使える操作性を目指す。
UC-ID ユースケース名 アクター 概要
UC-401 マニュアルを登録する 管理者 タイトル・YouTube URL・工程テキストを入力してマニュアルを新規作成できる（非エンジニアでも操作可能なUI）
UC-402 マニュアルを編集する 管理者 既存マニュアルのタイトル・URL・テキストを更新できる
UC-403 マニュアルを削除する 管理者 不要なマニュアルを削除できる
UC-404 カテゴリを管理する 管理者 カテゴリの追加・編集・削除ができる
UC-405 PDFを登録・更新する 管理者 マニュアルに紐付くPDFファイルをアップロード・差し替えできる
UC-406 習熟度ラベルを設定する 管理者 マニュアルに「初級・中級・上級」等の習熟度ラベルを付与できる

Phase 5 アカウント・権限管理

目的：ログイン機能と階層別アクセス制御を実装し、コンテンツの公開範囲を管理する。アカウント発行は初期運用期間中はCLI / GitHub Actionsで対応し、管理UIはエンハンスで追加する。
