# マニュアル閲覧UX（分割ビュー）プロトタイプ

## 検証目的

議事録の要件「YouTube動画とテキストが1画面で完結して閲覧できるUI」を検証する。

## 画面構成

| 画面 | URL | 役割 |
|------|-----|------|
| ブランド選択 | `index.html` | 業態を選ぶ |
| カテゴリ選択 | `index.html?brand=unagi` | カテゴリを選ぶ |
| マニュアル一覧 | `index.html?brand=unagi&category=eel-processing` | マニュアルを選ぶ |
| **ビューワー** | `viewer.html?brand=...&category=...&movie=...` | **動画＋テキスト閲覧（核心）** |

## レイアウト方針

### PC / タブレット（768px以上）
- 左カラム：YouTube iframe を sticky 表示（スクロールしても動画が追従）
- 右カラム：ステップリスト（クリックで選択中のステップ詳細を展開、動画も切り替わる）

### スマートフォン（767px以下）
- 動画（全幅）→ ステップ番号タブ（横スクロール）→ 現在ステップのテキスト詳細

## 検証ポイント

- [ ] PC/タブレットで動画とテキストを同時に見て作業できるか？
- [ ] スマートフォンで動画→スクロール→テキストの流れが自然か？
- [ ] ステップ切り替えで動画が対応ステップに変わることが分かるか？
- [ ] 警告・ポイントの視認性は十分か？
