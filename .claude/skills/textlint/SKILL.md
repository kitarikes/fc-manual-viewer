# textlint チェック

Markdownドキュメントの文章品質を textlint でチェックし、必要に応じて自動修正する。

## 実行手順

### 1. 対象ファイルを特定する

- 直前に Write/Edit した `.md` ファイルを対象にする
- ユーザーがファイルを指定した場合はそのファイルを対象にする
- 全体チェックを求められた場合は `npm run lint:text` を使う

### 2. textlint を実行する

```bash
# 特定ファイルのチェック
./node_modules/.bin/textlint <ファイルパス>

# docs全体のチェック
npm run lint:text
```

### 3. 結果に応じて対処する

| 状態 | 対応 |
|------|------|
| エラーなし | 「textlintチェック完了、問題なし」と報告して終了 |
| `✓` 付きエラーのみ | `--fix` で自動修正し、再チェックを実行する |
| `✓` なしエラーあり | エラー内容をユーザーに報告し、修正方針を確認する |

```bash
# 自動修正（✓マーク付きのエラーのみ）
./node_modules/.bin/textlint --fix <ファイルパス>

# 自動修正後に再チェック（残存エラーの確認）
./node_modules/.bin/textlint <ファイルパス>
```

### 4. 結果を報告する

修正した内容と残存エラーの有無をユーザーに伝える。
手動修正が必要なエラーがある場合は、エラー箇所と改善案を具体的に示す。
