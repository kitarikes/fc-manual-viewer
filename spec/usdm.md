## R1　ブランド選択機能

ユーザーは閲覧したいフランチャイズブランドを選択できる。

> **理由** — 複数ブランドのオペレーションマニュアルを一つのシステムで管理するため、閲覧開始前にブランドを特定する必要がある。

### R1-1　ブランド一覧をグリッド表示する

登録されているすべてのブランドをグリッド形式（auto-fill、最小幅 280px）で一覧表示する。各ブランドカードにはアイコン・名称・説明文を表示する。

> **理由** — 複数ブランドを視覚的に比較しやすくするため。

### R1-2　ブランドカードのホバー演出

ブランドカードにマウスオーバーした際に、カード上部にゴールドグラデーションのアクセントバーが表示され、カードが 3px 上方に移動し、ボーダーがゴールド色に変わる。

> **理由** — クリック可能であることを視覚的にフィードバックし、操作性を高めるため。

### R1-3　ブランド選択でカテゴリ画面へ遷移する

ブランドカードをクリックすると `?brand={slug}` のURLパラメータを付与してカテゴリ選択画面へ遷移する。

> **理由** — ブランド → カテゴリ → マニュアルの階層ナビゲーションを実現するため。

## R2　カテゴリ選択機能

ユーザーは選択したブランドに紐づくマニュアルカテゴリを一覧から選択できる。

> **理由** — ブランドごとに複数のオペレーションカテゴリ（下処理・仕込み・焼き等）が存在するため、目的のカテゴリに素早くアクセスできる必要がある。

### R2-1　ブランドに紐づくカテゴリのみを表示する

URLパラメータの `brand` スラッグを元に該当ブランドの ID を特定し、`brandId` が一致するカテゴリのみをフィルタリングして表示する。

> **理由** — 関係のないブランドのカテゴリが混在するとユーザーが混乱するため。

### R2-2　カテゴリカードにカテゴリ名・説明・マニュアル本数を表示する

カテゴリカードにはカテゴリ名（serif フォント）・説明文・マニュアル本数（または「準備中」）を表示する。

> **理由** — ユーザーが目的のカテゴリを識別し、コンテンツ量を事前に把握できるようにするため。

### R2-3　マニュアルが存在しないカテゴリは「準備中」と表示する

MOVIES データ内に該当 `categoryId` のマニュアルが 0 件の場合、マニュアル本数の箇所に「準備中」と表示する。

> **理由** — コンテンツが未整備であることをユーザーに明示し、空ページへの遷移による混乱を防ぐため。

### R2-4　カテゴリ選択でマニュアル一覧画面へ遷移する

カテゴリカードをクリックすると `?brand={brandSlug}&category={categorySlug}` のURLパラメータを付与してマニュアル一覧へ遷移する。

> **理由** — 選択したカテゴリのマニュアル一覧を閲覧できるようにするため。

## R3　マニュアル一覧機能

ユーザーは選択したカテゴリに属するマニュアルの一覧を閲覧し、目的のマニュアルを選択できる。

> **理由** — 1カテゴリに複数のマニュアルが存在するため、内容・難易度・所要時間を比較してから閲覧を開始できる必要がある。

### R3-1　選択カテゴリのマニュアル一覧を縦リスト表示する

`categoryId` が一致するマニュアルをフィルタリングし、縦方向のリストで表示する。マニュアルが 0 件の場合は「準備中」メッセージを表示する。

> **理由** — 複数のマニュアルを素早くスキャンできるようにするため。

### R3-2　マニュアルカードに必要情報を表示する

マニュアルカードには難易度バッジ・タイトル・説明文・所要時間（`estimatedMinutes`）・ステップ数（`totalSteps`）・タグ（`tags`）を表示する。

> **理由** — ユーザーがマニュアルの難易度・内容・所要時間を把握したうえで選択できるようにするため。

### R3-3　難易度を3段階で色分けバッジ表示する

| 難易度 | ラベル | 色 |
|--------|--------|----|
| `beginner` | 初級 | 緑 |
| `intermediate` | 中級 | オレンジ |
| `advanced` | 上級 | 赤 |

> **理由** — 視覚的に難易度を即座に識別できるようにするため。

### R3-4　マニュアルカード選択でビューワーへ遷移する

マニュアルカードをクリックすると `viewer/?brand={b}&category={c}&movie={id}` へ遷移する。

> **理由** — 選択したマニュアルの詳細閲覧（ビューワー）画面へ移動するため。

## R4　マニュアル閲覧機能（分割ビュー）

ユーザーはYouTube動画とステップテキストを1画面で同時に閲覧できる。

> **理由** — 現場スタッフが動画を見ながら手順テキストを同時確認できることで、立ち止まって操作する回数を減らし、学習効率を高めるため。

### R4-1　PC・タブレット（≥ 768px）は左右2カラムレイアウト

ビューポート幅 768px 以上の場合、画面を「動画ペイン（flex: 1）＋ステップペイン（幅 420px、タブレットは 320px）」の2カラムグリッドで表示する。高さはナビゲーションバー（52px）を除いた 100vh とし、両ペインとも独立スクロールする。

> **理由** — 大画面では動画とテキストを同時視認できることで、手を止めずに作業確認ができるため。

### R4-2　スマートフォン（< 768px）は縦1カラムレイアウト

ビューポート幅 768px 未満の場合、2カラムグリッドを解除し、動画ペインを上部に `position: sticky`（top: 52px）固定した縦1カラム表示に切り替える。ステップペインは非表示にし、代わりにモバイル用タブ＋ステップ詳細エリアを使用する。

> **理由** — 小画面では左右分割が困難なため、縦スクロールで動画 → タブ → テキストの自然な順序で閲覧できるようにするため。

### R4-3　ステップ選択で動画・詳細を連動更新する

ステップをクリックすると `selectStep(index)` が呼ばれ、以下を同時に更新する。

1. iframe の `src` を該当ステップの `videoId` に更新
2. ステップリストのアクティブ状態を切り替え
3. モバイルタブのアクティブ状態を切り替え（選択タブを中央スクロール）
4. モバイル詳細エリアを更新

> **理由** — ユーザーが任意のステップの動画とテキストを連動して確認できるようにするため。

## R5　ステップ詳細表示機能

ユーザーは各手順の説明文・習得ポイント・注意事項をステップ単位で確認できる。

> **理由** — 各手順の具体的な内容・重要ポイント・安全上の注意を明確に提示することで、スタッフの正確な業務習得を支援するため。

### R5-1　ステップヘッダーにステップ番号・タイトル・所要時間を表示する

ステップ番号（ゼロ埋め2桁）・タイトル・所要時間（分・秒表示）をステップアイテムのヘッダー行に表示する。

> **理由** — ユーザーが現在のステップ位置と所要時間を把握できるようにするため。

### R5-2　ステップの詳細説明文を表示する

アクティブなステップの `description` をステップ詳細エリアに表示する（sans-serif、行間 1.85）。

> **理由** — 手順の具体的な方法をテキストで補足し、動画だけでは伝わりにくい細部を説明するため。

### R5-3　習得ポイントをリスト表示する（keyPoints）

`keyPoints` 配列が 1 件以上の場合、「習得ポイント」ラベル（ゴールド色）の下にリスト形式で表示する。各項目の先頭にゴールドの丸点を付与する。

> **理由** — 重要な習得事項をユーザーが見落とさないよう、箇条書きで視覚的に強調するため。

### R5-4　注意事項を赤強調ボックスで表示する（warnings）

`warnings` 配列が 1 件以上の場合、赤いボーダー付きボックス内に「注意事項」ラベルと ⚠ アイコン付きリストで表示する。

> **理由** — 食品衛生・安全上の注意を見落とさないよう視覚的に強調するため。

### R5-5　PC用ステップアイテムはクリックで展開・折りたたみする

PC右ペインのステップアイテムは、クリックで説明・習得ポイント・注意事項の表示／非表示を切り替えるアコーディオン形式とする。アクティブ時はヘッダー背景が gold-pale に変わり、chevron が 180 度回転する。

> **理由** — ステップ一覧を常に視認しながら詳細を確認できるようにするため。

## R6　動画再生機能

ユーザーは各ステップに対応するYouTube動画を画面内で再生できる。

> **理由** — 動画によるビジュアル学習はテキスト単体より理解・定着率が高く、現場作業の再現性向上が期待されるため。

### R6-1　YouTube動画をiframeで埋め込み表示する

`videoId` が設定されているステップでは `https://www.youtube.com/embed/{videoId}?rel=0&modestbranding=1` を `src` に持つ iframe を表示する。iframe はビデオペインの残領域全体（`flex: 1`）を使用する。

> **理由** — 外部動画プラットフォームを活用することでコンテンツ管理コストを削減するため。

### R6-2　動画未設定時はプレースホルダーを表示する

`videoId` が未設定または `PLACEHOLDER` で始まる場合、iframe の代わりにプレースホルダー（再生アイコン＋「この手順の動画は準備中です」テキスト）を表示する。

> **理由** — 動画コンテンツ準備中であることを明示し、空白表示や読み込みエラーを防ぐため。

### R6-3　ステップ切替時に動画を即時更新する

ステップが切り替わるたびに `video-frame-wrap` の内容を新しい `videoId` でリビルドし、常にアクティブステップに対応する動画を表示する。

> **理由** — 選択したステップに対応する動画を即座に表示し、動画とテキストの対応関係を維持するため。

## R7　ナビゲーション機能

ユーザーはシステム内の現在位置を把握し、任意の上位画面へ戻ることができる。

> **理由** — 3段階の階層（ブランド → カテゴリ → マニュアル）を持つため、ユーザーが迷子にならないよう常に位置情報と戻るリンクを提供する必要がある。

### R7-1　ナビゲーションバーを全画面上部に固定表示する

高さ 52px のナビゲーションバーをすべての画面の最上部に `position: sticky` で表示する。左端に「戻る」リンク、中央にパンくずリスト、右端にアプリ名「FC Manual Viewer」を配置する。

> **理由** — どの画面・スクロール位置でもナビゲーションにアクセスできるようにするため。

### R7-2　パンくずリストで現在位置を表示する

| 画面 | パンくず |
|------|----------|
| カテゴリ選択 | スタッフポータル › ブランド名 |
| マニュアル一覧 | ブランド名 › カテゴリ名 |
| ビューワー | ブランド名 › カテゴリ名 › マニュアル名 |

最末端の項目は ivory 色で強調する。

> **理由** — ユーザーが現在の閲覧位置を瞬時に把握できるようにするため。

### R7-3　戻るリンクで上位画面へ遷移する

| 現在画面 | 戻り先 |
|----------|--------|
| ビューワー | マニュアル一覧 |
| マニュアル一覧 | カテゴリ選択 |
| カテゴリ選択 | スタッフポータル（外部） |

> **理由** — ユーザーが階層を辿って目的のコンテンツに素早くアクセスできるようにするため。

### R7-4　不正URLパラメータ時に上位画面へリダイレクトする

- `brand` パラメータが未指定または無効 → スタッフポータルへリダイレクト
- `brand` は有効だが `category` が無効 → カテゴリ選択画面へリダイレクト
- `movie` が無効 → エラー画面を表示

> **理由** — 不正なURLアクセスによる画面崩壊やエラー表示を防ぎ、ユーザーを安全な状態に誘導するため。

## R8　データモデル

システムはブランド・カテゴリ・マニュアル・ステップの4階層データを管理し、各画面に適切なデータを提供する。

> **理由** — システムが正確にコンテンツを管理・表示するためのデータ構造を定義する必要があるため。

### R8-1　ブランドデータ（BRANDS）を管理する

| フィールド | 型 | 説明 |
|------------|----|------|
| `id` | string | 識別子 |
| `name` | string | 表示名 |
| `slug` | string | URLスラッグ |
| `description` | string | 説明文 |
| `icon` | string | 絵文字アイコン |

> **理由** — 複数ブランドを識別・表示するために各ブランドの基本情報を保持するため。

### R8-2　カテゴリデータ（CATEGORIES）をブランドに紐づけて管理する

| フィールド | 型 | 説明 |
|------------|----|------|
| `id` | string | 識別子 |
| `brandId` | string | 所属ブランドID |
| `name` | string | カテゴリ名 |
| `slug` | string | URLスラッグ |
| `description` | string | 説明文 |
| `icon` | string | 絵文字アイコン |
| `movieCount` | number | マニュアル本数 |

> **理由** — ブランドごとのオペレーションカテゴリを整理し、ブランドフィルタリングを実現するため。

### R8-3　マニュアルデータ（MOVIES）をカテゴリに紐づけて管理する

| フィールド | 型 | 説明 |
|------------|----|------|
| `id` | string | 識別子 |
| `categoryId` | string | 所属カテゴリID |
| `title` | string | タイトル |
| `description` | string | 説明文 |
| `totalSteps` | number | ステップ数 |
| `estimatedMinutes` | number | 所要時間（分） |
| `difficulty` | string | `beginner` / `intermediate` / `advanced` |
| `tags` | string[] | タグ配列 |

> **理由** — 各カテゴリのマニュアル一覧表示と詳細情報の提供のため。

### R8-4　ステップデータ（SEQUENCES）をマニュアルに紐づけて管理する

| フィールド | 型 | 説明 |
|------------|----|------|
| `stepNumber` | number | ステップ番号 |
| `title` | string | タイトル |
| `videoId` | string | YouTube動画ID |
| `durationSeconds` | number | 動画秒数 |
| `description` | string | 説明文 |
| `keyPoints` | string[] | 習得ポイント配列 |
| `warnings` | string[] | 注意事項配列 |

SEQUENCES はマニュアル ID をキーとしたオブジェクト形式で管理する。

> **理由** — マニュアルの各手順詳細（動画・テキスト・注意事項）を表示するため。
